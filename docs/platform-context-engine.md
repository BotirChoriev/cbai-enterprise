# CBAI Platform Context Engine

**Version:** 1.0.0  
**Sprint:** Alpha 0.3 RC-2

## Purpose

The Platform Context Engine connects existing CBAI intelligence modules through one shared entity-aware context. It does **not** create new intelligence — it propagates current country, company, university, workspace, and search selections across routes.

## Architecture

```
URL search params (?country=&company=&university=&workspace=&q=)
        ↕
PlatformContextProvider (React — single source of truth)
        ↕
buildPlatformContext() ← entity registries + context-history (localStorage)
        ↕
PlatformContextHeader → breadcrumbs, quick actions, related modules
        ↕
Entity pages (Countries / Companies / Universities) read + write context
```

### Module layout (`lib/context/`)

| File | Role |
|------|------|

## Context parameters

| Param | Meaning |
|-------|---------|
| `country` | Current country registry ID |
| `company` | Current company registry ID |
| `university` | Current university registry ID |
| `workspace` | Active workspace (`government`, `investor`, `citizen`) |
| `q` | Global search query preserved across navigation |

## Entity lifecycle

1. **Select** — User picks entity on Countries / Companies / Universities route.
2. **Record** — Provider writes entity ID to URL and appends to recent history (localStorage).
3. **Propagate** — All intelligence routes read the same snapshot via `usePlatformContext()`.
4. **Navigate** — Related module links include current context query string.
5. **Timeline** — Architecture reserved; displays *"No verified evidence timeline connected."*

## Cross-module navigation

When a country is active (e.g. Japan), related modules include:

Countries → Evidence → Reports → Investor → Government → Reasoning → Knowledge Graph

Company context adds: Companies → Countries → Investor → Evidence → Reports

University context adds: Universities → Countries → Evidence → Reports → Knowledge Graph

All links are generated by `buildContextualHref()` — no duplicated state in module code.

## Routes with context header

- `/countries`, `/companies`, `/universities`
- `/knowledge`, `/analytics`, `/reasoning`
- `/government`, `/investor`, `/citizen`
- `/search`, `/graph`

- No scores, AI claims, or hidden automation
